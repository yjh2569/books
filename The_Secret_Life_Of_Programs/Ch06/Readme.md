# 입출력과 네트워킹

## 네트워킹

* 근거리 네트워크(LAN) : 집이나 사무실 같이 좁은 지리적 영역을 묶는 네트워크
* 광역 네트워크(WAN) : 더 넓은 지리적 영역을 묶는 네트워크
* 인터넷 : 네트워크들로 이루어진 네트워크. 즉, 여러 LAN을 하나로 연결해주는 WAN

### 최근의 LAN

* 최근에는 이더넷(Ethernet)이라 불리는 LAN이 주로 사용되고 있다.
* 각 장치의 네트워크 인터페이스에 MAC 주소라는 유일한 48비트 주소가 부여된다.
* 데이터는 약 1500바이트의 프레임이라는 패킷으로 구성된다.
* LAN 내 모든 장치가 같은 선에 연결되어 있어 어떤 장치에서 보낸 데이터가 모든 장치에 전달된다. 다만 프레임의 목적 MAC 주소가 자신의 MAC 주소와 일치하지 않는 장치들은 데이터를 무시한다.
* 둘 이상의 장치가 동시에 데이터를 전송하는 경우 충돌이 발생할 수 있다. 이러한 경우 랜덤 백오프 후 재시도(random back-off-retry)를 통해 재전송한다.

### 인터넷

* 여러 계층으로 이뤄진 프로토콜의 집합
* 프로토콜 계층은 물리적 네트워크를 정의하는 아래쪽 계층을 변경해도 위쪽 계층에 영향을 끼치지 않도록 고안됐다.
* TCP/IP : 인터넷이 사용하는 두 가지 프로토콜
  * IP : 패킷(데이터그램, datagram)을 한 곳에서 다른 곳으로 옮겨준다.
  * TCP : IP 위에 만들어져 패킷이 제대로 전달되었는지를 확실히 보장한다.
* IP 주소 : 인터넷 상에서 각 컴퓨터에 할당된 유일한 주소
  * 하드웨어에 묶여 있지 않아 변경될 수 있다.
  * 인터넷은 대부분 IPv4에서 이루어진다. 다만 32비트 주소를 사용해서 현재 할당할 수 있는 주소가 남아 있지 않다. 그래서 점점 128비트를 사용하는 IPv6 쪽으로 나아가고 있다.
* 도메인 이름 시스템(DNS) : 도메인 이름을 IP 주소로 바꿔주는 시스템
* 월드 와이드 웹
  * SMTP : 단순 우편 전달 프로토콜로 전자우편 시스템이 작동하는 기반 프로토콜
  * HTTP : 하이퍼텍스트 전송 프로토콜로 가장 많이 사용하는 프로토콜이다. 보안이 강화된 HTTPS 프로토콜도 있다.
  * 하이퍼텍스트 : 링크가 붙은 텍스트
  * HTTP는 웹 브라우저와 웹 서버가 상호작용하는 방법을 정의한다.
  * 웹 브라우저는 웹 페이지를 볼 때 사용하는 프로그램, 웹 서버는 요청한 페이지를 제공하는 서버다.
  * URL : 웹 브라우저 주소창에 입력함으로써 원하는 웹 페이지를 얻을 수 있다.
  * 웹 페이지는 보통 HTML로서 존재한다.
